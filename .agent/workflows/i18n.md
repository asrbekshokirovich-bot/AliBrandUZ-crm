# /i18n — Translation Workflow

Manage translations for alicargo-joy-main. All UI text must be in 3 languages: Uzbek (uz), Russian (ru), English (en).

## Translation File Location
```
src/i18n/config.ts   ← All translations live here
```

## How to Add a New Translation Key

### Step 1: Find the right section in config.ts
```typescript
// config.ts structure:
const resources = {
  uz: { translation: { /* uz keys */ } },
  ru: { translation: { /* ru keys */ } },
  en: { translation: { /* en keys */ } },
};
```

### Step 2: Add the key to ALL 3 languages simultaneously
```typescript
// uz:
nakladnoy_marketplace: 'Platforma tanlang',

// ru:
nakladnoy_marketplace: 'Выберите платформу',

// en:
nakladnoy_marketplace: 'Select Marketplace',
```

### Step 3: Use in component
```typescript
const { t } = useTranslation();
<span>{t('nakladnoy_marketplace')}</span>
```

## Common Naming Conventions
```
box_*        → Box/Qutilar related
mp_*         → Marketplace related
tash_*       → Tashkent dashboard
ai_*         → Ali AI Brain
crm_*        → CRM general
auth_*       → Auth/Login
nav_*        → Navigation
```

## Missing Translation Check
```bash
# Search for hardcoded Uzbek text that should be translated
grep -r "O'zbekiston\|tovar\|Toshkent" src/pages/ --include="*.tsx"
```

## Usage
```
/i18n "add translation for new marketplace labels"
/i18n "fix missing Russian translations in Boxes page"
```
